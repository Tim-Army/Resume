# Bash automation

## Current

- `lint-resume.sh` — Enforces the two mechanical editorial rules across
  `resume/**/*.md`. Run it directly, or let CI run it via
  `.github/workflows/lint-resume.yml`.

```sh
scripts/bash/lint-resume.sh          # check every resume source
scripts/bash/lint-resume.sh FILE...  # check specific files
```

It exits non-zero on any violation and is safe to run at any time.

**Everything else in this directory is historical.** None of it is the current
build path: the resume PDF is generated by
`scripts/python/create-full-resume-pdf.py`. See
[docs/resume-build-manifest.md](../../docs/resume-build-manifest.md) for the
current build, and the repository [README](../../README.md) for the command.

## Historical scripts

These scripts were written as one-shot tasks: each hardcodes an absolute
repository path and a single commit message, and several embed a full copy of
the resume text in a heredoc. Those embedded copies are stale —
`resume/master/Tim-Fox-Resume.md` is the source of truth.

Some have been kept superficially current through bulk find-and-replace (for
example when the PDFs were renamed), which makes them look maintained without
making them safe to run.

The `create-one-page-resume-and-push*.sh` family generated the concise resume,
which was retired on 2026-07-19. They no longer have a live output.

Read them for reference. Do not run them.

## Policy

- Resume content belongs in `resume/*.md`, never in a script heredoc.
- Do not create `-v2`, `-fixed`, or timestamped copies of a script.
- Do not run permanent repository scripts from `~/Downloads`.
- Run `bash -n scripts/bash/<script>.sh` before execution.
- Make committed scripts executable with `chmod +x`.
- Do not store credentials, tokens, private keys, or controlled information in scripts.
